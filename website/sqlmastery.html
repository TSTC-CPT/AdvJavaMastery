MySQL Mastery Assignment

For this assignment you will be setting up tables in your own MySQL database. Use the schema and login information provided by your instructor. Information in your program will now need to be saved and retrieved from the remote database. This includes all account information, lodges, and images. Every member variable in the Employee, Customer,and Lodge class must be in the database tables. Use a single/shared connection variable for all of your database operations. Open the connection at the beginning of database operations and close the connection when the operations for a set of data are complete. Include the SQL driver .jar file in your project folder and link it accordingly using a relative path. Files will now serve as backup in case of database or connection failure. First, attempt to use the database, if the connection fails, resort to using the local files.

Images will need to be incorporated into the program for the lodges. The Customer and Employee views will display several images for each lodge. Provide a preview area using a JScrollPane or JPanel with a FlowLayoutopens in new window. The preview viewport must show a least 3 images. Reduced to the preview size of the image to a thumbnail. If a user clicks on an image, a large window will open that will allow the user to cycle through large scale versions of the images. Add an ArrayList to your Lodge class to store the images for each lodge. Load 2 Houses and 2 Hotels into your database. Include at least 7 4k images for 1 lodge. The other lodges only need 3 images each.

The customers orders will also need a table in the database. When a customer places an order, store all relevant information as a row in this table so it can be used for collecting order history. The GUI side of your interface is required to use the MM/DD/YYYY format for all dates. Make conversions as needed to interact with the database.

The Employee view will allow the user to add images to a lodge. Use the JFilechooseropens in new window to select image files and allow the user to select multiple image files at once. Set a filter to make locating PNG and JPG images easier. The images will be stored in the database. Create a separate table for the images and link them to the lodges with a foreign key. Make sure each lodge can support as many images as the employee wants to add. Verify that additions and edits to the lodges are updated in the Employee and Customer views. When a lodge is removed, delete the images linked to it as well.

The Manager view will also be altered to work with the database. Store all employee information in a table in the database. All add, edit and remove operations to employees with affect this table. Refresh the GUI when changes are made to the table. Add reports to the Manager view as specified below.

Add HTML reports to your program. Use CSS to add color and style to your reports in HTML tableopens in new window format. Customer reports will show order history for a selectable time frame for the current user. Inform guests that they will need to create an account to have access to order history. Employee reports will list all customers and their total spending for a selectable date range. Manager reports will list total revenue for all lodges over a selectable date range. All time frames will be selected with date selector comboboxs. Open the report in the default browser and save a copy of the HTML file in a local folder called “Reports”. Store the Reports folder in the user's Documents folder (Windows OS).

Optimize your program by reducing the number of calls to the database to as few as possible. Add a refresh button to the Customer, Employee and Manager views to reload and refresh the table data from the database. This can be helpful for debugging or verifing information if the connection fails. Only reload the data relevant to the current view. Keep all of your table creation and startup data scripts. Add the scripts to your project folder in a single text file.
