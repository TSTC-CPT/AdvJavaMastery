Swing Mastery

For this assignment you will be moving the existing program into a GUI format using Java Swing. Each state of the program will have at least 1 window/form to operate out of. Try to arrange the components on the forms in a way that is organized and professional. Keep all of the forms the same size and in the same location on the screen.

Login
Use the JLabel to label the inputs for name and password. Use JTextField or JPasswordField for input. Always mask the password field. This should look like a typical login form with JButtonsopens in new window to Register, Login, Continue as Guest, or Exit. You may create a separate form for registering or incorporate that function into the login form. When the user is registering a new account provide real time updates on the strength and validity of the password using a KeyListener. You can use a label such as ‘must be x characters long’ or you may want to change the color of the field to red to indicate the password is insufficient. The indicator must be dynamic and change in real time as the password text is modified.

Customer View
The customer view will provide a list of lodges read in from the file created in the previous assignment. There are several design options here, but the user must be able to scroll through the different lodges so a JScrollPane must be implemented. You may want to combined it with a JList or JTable to list the names of the lodges. All lodge information must be retrieved from the file for each lodge and displayed when that lodge is selected. Pictures will be added for each lodge in a later assignment so leave some room for a thumbnail viewing area. From this point the customer will need to be able to select a lodge and a date range for their stay. Use the calendar from the practice assignment as your date selection tool.

Employee View
Transfer the functionality of the console based program into a swing application. Use the JScrollPane component with a JList or JTable to display the file contents. The employee will have the option to add pictures to the selected lodge in the next assignment so allow some room for that in your design. Use JButtons or a menuopens in new window system to add, edit and remove lodges from the program. Use a JTabbedPaneopens in new window with tabs to separate the manager view from the employee view. Disable or hide the manager tab when user is not a manager.

Manager View
The manager view will also implement everything in the console program. No pictures will be needed here. Use a JTable and JScrollPane to organize and modify employee information. The information will need to be loaded and saved to the file. You may want to use JButtons or add a menuopens in new window system to implement these functions in your program. Managers will be able to move freely between the employee view and the manager view by clicking on the corresponding tab.

Refinement
Try to add polish to your program by making it robust and user friendly. Use labels or a non-editable text area to update the user of the current status of the program. If an employee adds a new lodge successfully it may be good to clear the fields for the next entry (depending on your design) Avoid using annoying popup message boxes to give status updates. Validate the user’s input and disable buttons that should not be clicked until there is valid input to work with. Add color to your program. Use a pleasant pallet with good contrast. Check out coolorsopens in new window for some good ideas and easy to use copy/paste color schemes.
