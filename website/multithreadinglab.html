Java Multithreading Lab

Overview
Write a program that takes advantage of multithreading to sort a large array of random numbers. 40,000 numbers may be a good starting place. Measure the time efficiency of multithreading versus a single thread implementation. Execute the efficiency test 10 times for each sort and store the time it takes for each pass in an array. Make sure your array size can be changed in a single place within the code. Do not hardcode the array size, rather use a final variable to set the size.

Sorting and Threading
Use the Bubble Sort to sort the array of random numbers. (Adjust as needed for your hardware) Repeat the sort using a copy of the same array but use 2 threads, half of the array to each thread. Then merge the 2 halves by writing a merge algorithm. You can use a new array to store the result. Measure the time difference between multithreaded sorting and single thread sorting. Repeat this process 10 times with 10 different arrays and log the time difference in a text file. Label each output in the file. Calculate the average single sort time, average threaded sort time and the average difference as a percent. Write these 3 values to the end of the file with a label.

System.nanoTime() can be used for the timer. Set the time before the sort, and then subtract that time from the current time after the sort is complete. The goal is to determine how much faster a 2 thread bubble+merge sort is, compared to a single thread bubble sort on the same array of numbers. If the program is working correctly, you should see X4 speed improvement using 2 threads (on a large array). Use a constant int to set the array size and adjust it so that the finish time is around a few seconds.

Tips on the merge: For this part of the program you only need to compare the values of both sorted array halves and take the lowest value. Then go to the next number in that array and compare again. The values you are comparing will seldom be the same index so you will need 2 different variables to keep track of the indexes. Convert your times stamp output to something more meaningful such as seconds or milliseconds.

Example: 

                Array1 (sorted)     Array2 (sorted)
                1                   3 
                3                   3
                4                   6
                7                   8
                
                Merged Result: 1 3 3 3 4 6 7 8
