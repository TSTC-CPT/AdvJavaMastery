Hotel Booking Application
Java Group Project

Overviewüî≠
Create an application that simulates a hotel booking service. The app will have 2 views, one for the customer and one for the booking agency. Customers will be able to select dates, locations, and book hotel reservations. Employees of the booking agency will be able to view bookings, update hotel information and check revenue generated by customers using the app.

Employee Viewüë®‚Äçüíº
The employee view will be accessed when a user logs in with an employee account. This will be the only thing that determines which view is used. No check boxes or other indicators will be used to access the employee view. Employees will be able to:

  • Add, remove and update hotels in the system
  • View all bookings
  • View all revenue over a specified date range (a small percentage of all bookings)
  • View customer information including total spending

Customer ViewüôÇ
Customers will be able to choose a date range and a location. Dates must be selectable from a calendar GUI. Customers will then be presented with a listing of hotels available in that location. Hotel listings should include a brief description, ratings, pricing, room availability and a few pictures representing that hotel. Customers will also need the option to create an account or sign in. When a customer has logged in they will be able to see their currently booked hotels and account history. From account history a customer will be able to rate a previous hotel stay.

DatabaseüóÇÔ∏è
Data will be stored on a remote MySQL database. An ERD should be drawn up as an early priority. Hotel information, bookings, customer accounts and employee accounts should all be stored on the database. The database should be populated with 3 customer accounts including account history and at least 1 employee. The names are c1, c2, c3 and e1, and the password must be password for all of these accounts. Use these accounts to thoroughly test your program and demonstrate its functionality when graded.

Hotel Dataüè®
You will need at least 10 distinct hotels and 5 locations. A location must have at least 3 but no more than 8 hotels. Some hotels may be used in multiple locations (hotel chain). These hotels may use the same name, images and descriptions but room availability, ratings and pricing should be independent. Hotel names and locations may be real or fictional.

Java does not include a date or date range selection tool so it will need to be created from swing components and Java‚Äôs GregorianCalendar class.
