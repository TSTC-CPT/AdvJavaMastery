Polymorphism Mastery Assessment v1.1

Continue building and modifying the travel agency program by updating your previous assignment. If any class doesn’t need need to be instantiated then make it abstract. Override the toString() method in each class. Any object represented by one of the classes must be created and used as an object from that class (Customers, Hotels, etc).

Setup a polymorphic state machine to implement these 4 states of the program. Each state will operate out of a menu as shown below.

  • Customer View
  • Employee View
  • Manager View
  • Login
Login State
  1. Login
  2. Register
  3. Continue as Guest
  4. Exit

Login will prompt the user for a name and password. If the name and password match the registered name and password, the user will proceed to the Customer View. You will need to start the program with 1 employee and 1 manager. There names are Employee and Manager and their password is password. If the user logs in as either of these, the program will shift to the corresponding state.

Registry will keep track of customers in an array list. More permanent storage will be incorporated in future assignments, but for now the object data will only be held in memory. The user will be prompted for a name and password. Validate the password. Require a specific length, at least 1 capital letter, and 1 special character. Any new account will be a Customer.

Continue as Guest will move straight to the Customer View.

Employee View
  1. Add Lodge
  2. Remove Lodge
  3. List Lodges
  4. Log out
  5. Manager View (only visible if logged in as a manager)

Use an array list to maintain a list of lodges. Populate the list with at least 2 Lodges from each subclass. Implement all of the functionality listed in the menu above.

Manager View
  1. Add Employee
  2. Remove Employee
  3. List Employee
  4. Make Employee A Manager
  5. Manage Lodges (Switch to Employee View)
  6. Log out

From the Manager View the user will be able to add and remove employees as well as promote regular employees to a manager position. Managers also have the ability to switch to the standard Employee View to manage the Lodges in the program. The program will need an array list with 2 employees preloaded into the program as mentioned above.

Customer View
  1. View Lodges with price per night
    1. Select Lodge
      1. Enter number of days
    2. Return to previous menu
  2. View Order
    1. Confirm Order
    2. Cancel Order
    3. Return
  3. Exit

The customer will have 3 basic options as listed above. View Lodges will provide a number list of lodge names followed by the price per night. Selecting a lodge will then prompt for the number of nights. After the customer enters the number of nights, return to the main menu where they will have the option to view the order and total cost. Here they will have the options to confirm or cancel. Either option will clear the cart and provide an appropriate message. You may want to create an Order class to help manage this data.

Modifications to main()

Remove the test code from the previous assignment and setup a loop for your state machine. Create all of the objects for the states and start the program in the Login state.
