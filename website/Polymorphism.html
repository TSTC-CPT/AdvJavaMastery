Continue building and modifying the travel agency program by updating your previous assignment. If any class doesn’t need to be instantiated then make it abstract. Any object represented by one of the classes must be created and used as an object from that class (Customers, Hotels, etc). Do not use Strings to represent these objects. There are several menus in this program. Always use numbered menu options for input and take user input as a String. Use nextLine() rather than next() to reduce input errors. Do not ask the user to type whole words when prompting for things like House or Hotel or true/false. Instead use single key menus such as:
1. House

Setup a polymorphic state machine to implement these 4 states of the program. Use a separate java file for each state class. Each state will operate out of a menu as shown below.

  • Customer View
  • Employee View
  • Manager View
  • Login
Login State
  1. Login
  2. Register
  3. Continue as Guest
  4. Exit

Login will prompt the user for a name and password. If the name and password match a registered name and password, the user will proceed to the Customer View. You will need to start the program with 1 employee and 1 manager. Their names are Employee and Manager and their password is password. If the user logs in as either of these, the program will shift to the corresponding state. These 2 names and passwords are required to work for all future modules in this course.

Registry will keep track of customers in an array list. More permanent storage will be incorporated in future assignments, but for now the object data will only be held in memory. The user will be prompted for a name and password. Validate the password. Require a minimum length, at least 1 capital letter, and 1 special character. Any new account will be a Customer.

Continue as Guest will move straight to the Customer View.

Employee View
  1. Add Lodge
  2. Remove Lodge
  3. List Lodges
  4. Log out
  5. Manager View (only visible if logged in as a manager)

Use an ArrayList to maintain a list of lodges. This ArrayList will need to be accessible from both the Employee view and the Customer view. You can choose between using a single ArrayList for all lodges or 2 separate lists for houses and hotels. These ArrayList must contain objects from your classes, not Strings. When a user selects 'Add Lodge', prompt the user for all of the data for the type of lodge they choose. Set all member variables for the new Lodge and then add the lodge to the ArrayList. Hardcode the list with at least 2 Lodges from each subclass. Implement all of the functionality listed in the menu above. Number any list you display for easy selection and a professional appearance.

Manager View
  1. Add Employee
  2. Remove Employee
  3. List Employees
  4. Make Employee A Manager
  5. Manage Lodges (Switch to Employee View)
  6. Log out

From the Manager View the user will be able to add and remove employees as well as promote regular employees to a manager position. Use the isManager boolean from the Employee class to determine if the user is a manager. Managers also have the ability to switch to the standard Employee View to manage the Lodges in the program. The program will need an ArrayList with 2 employees preloaded into the program as mentioned above. Use numbers when displaying the list for easy selection. Manager's are not required to follow the password rules. A manager may choose any password when adding a new employee.

Customer View
  1. View Lodges with price per night
    1. Select Lodge
      1. Enter number of days
    2. Return to previous menu
  2. View Order
    1. Confirm Order
    2. Cancel Order
    3. Return
  3. Return

The customer will have 3 basic options as listed above. View Lodges will provide a numbered list of lodge names followed by the price per night. Selecting a lodge will then prompt for the number of nights. After the customer enters the number of nights, return to the main menu where they will have the option to view the order and total cost. Here they will have the options to confirm or cancel. Either option will clear the cart and provide an appropriate message. You may want to create an Order class to help manage this data.
